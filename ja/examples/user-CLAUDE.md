# ユーザーレベル CLAUDE.md の例

これはユーザーレベルの CLAUDE.md ファイルの例です。`~/.claude/CLAUDE.md` に配置してください。

ユーザーレベルの設定はすべてのプロジェクトにグローバルに適用されます。以下の用途に使用してください:
- 個人的なコーディングの好み
- 常に適用したい普遍的なルール
- モジュラールールへのリンク

---

## 基本理念

あなたは Claude Code です。複雑なタスクには専門エージェントとスキルを使用します。

**主要な原則:**
1. **エージェント優先**: 複雑な作業は専門エージェントに委譲する
2. **並列実行**: 可能な場合は Task ツールで複数のエージェントを使用する
3. **実行前に計画**: 複雑な操作にはプランモードを使用する
4. **テスト駆動**: 実装前にテストを書く
5. **セキュリティ最優先**: セキュリティに妥協しない

## 個人設定

### コードスタイル
- コード、コメント、ドキュメントに絵文字を使用しない
- イミュータビリティを優先 - オブジェクトや配列を直接変更しない
- 大きなファイルよりも小さなファイルを多く
- 通常200〜400行、最大800行

### Git
- コンベンショナルコミット: `feat:`, `fix:`, `refactor:`, `docs:`, `test:`
- コミット前に必ずローカルでテストする
- 小さく焦点を絞ったコミット

### テスト
- TDD: テストを先に書く
- 最低80%のカバレッジ
- 重要なフローにはユニット + 統合 + E2E

## 成功指標

以下を満たした場合に成功とみなします:
- すべてのテストが通過（80%以上のカバレッジ）
- セキュリティ脆弱性がない
- コードが読みやすく保守しやすい
- ユーザー要件を満たしている

---

**理念**: エージェント優先設計、並列実行、行動前に計画、コード前にテスト、常にセキュリティ。
