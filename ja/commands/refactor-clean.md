# リファクタリング・クリーン

テスト検証付きでデッドコードを安全に特定・削除：

1. デッドコード分析ツールを実行：
   - knip：未使用のエクスポートとファイルを検出
   - depcheck：未使用の依存関係を検出
   - ts-prune：未使用の TypeScript エクスポートを検出

2. 包括的なレポートを .reports/dead-code-analysis.md に生成

3. 検出結果を重要度別に分類：
   - 安全：テストファイル、未使用のユーティリティ
   - 注意：API ルート、コンポーネント
   - 危険：設定ファイル、メインエントリーポイント

4. 安全な削除のみを提案

5. 各削除の前に：
   - 全テストスイートを実行
   - テストが通ることを確認
   - 変更を適用
   - テストを再実行
   - テストが失敗した場合はロールバック

6. クリーンアップ済みアイテムのサマリーを表示

テストを実行せずにコードを削除しないこと！
