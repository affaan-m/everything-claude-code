# ドキュメント更新

信頼できる情報源からドキュメントを同期：

1. package.json の scripts セクションを読み込む
   - スクリプトのリファレンステーブルを生成
   - コメントからの説明を含める

2. .env.example を読み込む
   - すべての環境変数を抽出
   - 目的とフォーマットを記載

3. docs/CONTRIB.md を生成（以下を含む）：
   - 開発ワークフロー
   - 使用可能なスクリプト
   - 環境構築
   - テスト手順

4. docs/RUNBOOK.md を生成（以下を含む）：
   - デプロイ手順
   - モニタリングとアラート
   - よくある問題と解決方法
   - ロールバック手順

5. 古くなったドキュメントを特定：
   - 90日以上更新されていないドキュメントを検出
   - 手動レビュー用にリスト化

6. 差分のサマリーを表示

信頼できる情報源：package.json と .env.example
