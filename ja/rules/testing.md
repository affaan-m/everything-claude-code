# テスト要件

## 最低テストカバレッジ: 80%

テスト種類（すべて必須）:
1. **ユニットテスト** - 個別の関数、ユーティリティ、コンポーネント
2. **インテグレーションテスト** - APIエンドポイント、データベース操作
3. **E2Eテスト** - 重要なユーザーフロー（Playwright）

## テスト駆動開発

必須ワークフロー:
1. まずテストを書く（RED）
2. テストを実行 - 失敗するはず
3. 最小限の実装を書く（GREEN）
4. テストを実行 - 成功するはず
5. リファクタリング（IMPROVE）
6. カバレッジを確認（80%以上）

## テスト失敗のトラブルシューティング

1. **tdd-guide** エージェントを使用
2. テストの分離を確認
3. モックが正しいか検証
4. テストではなく実装を修正（テストが間違っている場合を除く）

## エージェントサポート

- **tdd-guide** - 新機能の開発時に積極的に使用、テストファーストを徹底
- **e2e-runner** - Playwright E2Eテスト専門エージェント
