# 用户级 CLAUDE.md 示例

这是一个用户级 CLAUDE.md 文件示例。放置在 `~/.claude/CLAUDE.md`。

用户级配置在所有项目中全局生效。用于：
- 个人编码偏好
- 你始终希望执行的通用规则
- 链接到你的模块化规则

---

## 核心理念

你是 Claude Code。我使用专门的代理和技能来处理复杂任务。

**关键原则：**
1. **代理优先**：将复杂工作委托给专门的代理
2. **并行执行**：尽可能使用 Task 工具配合多个代理
3. **执行前规划**：复杂操作使用计划模式
4. **测试驱动**：在实现之前编写测试
5. **安全第一**：绝不在安全上妥协

## 个人偏好

### 代码风格
- 代码、注释或文档中不使用表情符号
- 优先使用不可变性 - 绝不修改对象或数组
- 多个小文件优于少数大文件
- 通常 200-400 行，每个文件最多 800 行

### Git
- 约定式提交：`feat:`、`fix:`、`refactor:`、`docs:`、`test:`
- 提交前始终在本地测试
- 小而专注的提交

### 测试
- TDD：先写测试
- 最低 80% 覆盖率
- 关键流程需要单元 + 集成 + E2E 测试

- Agent Panel 用于文件跟踪

---

## 成功标准

当以下条件满足时，你是成功的：
- 所有测试通过（80%+ 覆盖率）
- 无安全漏洞
- 代码可读且可维护
- 满足用户需求

---

**理念**：代理优先设计，并行执行，先计划后行动，先测试后编码，安全始终如一。
