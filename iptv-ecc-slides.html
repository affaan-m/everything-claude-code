IPTV 개발 적용 02 / 02

기획 → 개발 → 검증
전 사이클 커버 방법

각 단계에 맞는 커맨드·에이전트를 조합해 반복 가능한 워크플로우를 구성한다

📋
기획 · 설계
/prp-prd 요구사항 → PRD 자동 작성
/prp-plan BE/FE/DB 작업으로 분해
/plan 구현 전략 수립 · 검토
planner agent 복잡한 기능 아키텍처 설계
architect agent 시스템 구조 결정 지원
IPTV 예시 "VOD 재생 이력 기능" 요청
→ PRD 작성 → API 스펙 · DB 스키마 · UI 컴포넌트로 자동 분해
⚡
개발 · 구현
/multi-backend 스트리밍 API · EPG · 과금 병렬 개발
/multi-frontend 플레이어 UI · 채널 가이드 병렬 개발
/prp-implement 계획을 실제 코드로 구현
/build-fix 빌드 오류 자동 진단·수정
skill-create 기존 코드에서 팀 스킬 자동 추출
IPTV 예시 HLS 스트림 API (BE) +
비디오 플레이어 컴포넌트 (FE)
→ 병렬 개발로 리드타임 단축
🛡️
검증 · 배포
/tdd TDD 워크플로우 · 단위 테스트
/e2e E2E 테스트 생성 · 실행
/quality-gate 커밋 전 품질·보안 체크
/code-review 보안·성능·DB 전문 에이전트 리뷰
/prp-pr PR 자동 생성 · 제출
IPTV 예시 DRM 인증 로직 보안 리뷰 →
재생 흐름 E2E 테스트 →
구독 API 성능 최적화 검토
