# /learn - 提取可复用模式

分析当前会话并提取值得保存为技能的模式。

## 触发时机

在会话中解决了非平凡问题时，随时运行 `/learn`。

## 提取内容

寻找以下内容：

1. **错误解决模式**
   - 发生了什么错误？
   - 根本原因是什么？
   - 什么修复了它？
   - 这对类似错误是否可复用？

2. **调试技巧**
   - 非显而易见的调试步骤
   - 有效的工具组合
   - 诊断模式

3. **变通方案**
   - 库的特殊行为
   - API 限制
   - 特定版本的修复

4. **项目特定模式**
   - 发现的代码库约定
   - 做出的架构决策
   - 集成模式

## 输出格式

在 `~/.claude/skills/learned/[模式名称].md` 创建技能文件：

```markdown
# [描述性模式名称]

**提取日期：** [日期]
**上下文：** [简要描述何时适用]

## 问题
[这解决什么问题 - 要具体]

## 解决方案
[模式/技术/变通方案]

## 示例
[如适用，提供代码示例]

## 使用场景
[触发条件 - 什么应该激活此技能]
```

## 流程

1. 回顾会话中可提取的模式
2. 识别最有价值/可复用的见解
3. 起草技能文件
4. 在保存前请用户确认
5. 保存到 `~/.claude/skills/learned/`

## 注意事项

- 不要提取平凡的修复（拼写错误、简单语法错误）
- 不要提取一次性问题（特定的 API 中断等）
- 专注于能在未来会话中节省时间的模式
- 保持技能聚焦 - 每个技能一个模式
