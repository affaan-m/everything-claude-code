# Eval 命令

管理评估驱动的开发工作流。

## 用法

`/eval [define|check|report|list] [功能名称]`

## 定义评估

`/eval define 功能名称`

创建新的评估定义：

1. 创建 `.claude/evals/功能名称.md` 文件，使用模板：

```markdown
## EVAL：功能名称
创建日期：$(date)

### 能力评估
- [ ] [能力描述 1]
- [ ] [能力描述 2]

### 回归评估
- [ ] [现有行为 1 仍然正常工作]
- [ ] [现有行为 2 仍然正常工作]

### 成功标准
- 能力评估的 pass@3 > 90%
- 回归评估的 pass^3 = 100%
```

2. 提示用户填写具体标准

## 检查评估

`/eval check 功能名称`

运行功能的评估：

1. 从 `.claude/evals/功能名称.md` 读取评估定义
2. 对于每个能力评估：
   - 尝试验证标准
   - 记录通过/失败
   - 在 `.claude/evals/功能名称.log` 中记录尝试
3. 对于每个回归评估：
   - 运行相关测试
   - 与基准进行比较
   - 记录通过/失败
4. 报告当前状态：

```
评估检查：功能名称
========================
能力：X/Y 通过
回归：X/Y 通过
状态：进行中 / 就绪
```

## 评估报告

`/eval report 功能名称`

生成全面的评估报告：

```
评估报告：功能名称
=========================
生成日期：$(date)

能力评估
----------------
[评估-1]：通过 (pass@1)
[评估-2]：通过 (pass@2) - 需要重试
[评估-3]：失败 - 查看备注

回归评估
----------------
[测试-1]：通过
[测试-2]：通过
[测试-3]：通过

指标
-------
能力 pass@1：67%
能力 pass@3：100%
回归 pass^3：100%

备注
-----
[任何问题、边缘情况或观察结果]

建议
--------------
[可发布 / 需要改进 / 阻塞]
```

## 列出评估

`/eval list`

显示所有评估定义：

```
评估定义
================
feature-auth      [3/5 通过] 进行中
feature-search    [5/5 通过] 就绪
feature-export    [0/4 通过] 未开始
```

## 参数

$ARGUMENTS：
- `define <名称>` - 创建新的评估定义
- `check <名称>` - 运行并检查评估
- `report <名称>` - 生成完整报告
- `list` - 显示所有评估
- `clean` - 清除旧的评估日志（保留最近 10 次运行）
