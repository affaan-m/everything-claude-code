# Evalコマンド

評価駆動開発ワークフローを管理します。

## 使用方法

`/eval [define|check|report|list] [feature-name]`

## 評価の定義

`/eval define feature-name`

新しい評価定義を作成:

1. `.claude/evals/feature-name.md`をテンプレートで作成:

```markdown
## EVAL: feature-name

作成日: $(date)

### 機能評価

- [ ] [機能1の説明]
- [ ] [機能2の説明]

### リグレッション評価

- [ ] [既存の動作1がまだ機能する]
- [ ] [既存の動作2がまだ機能する]

### 成功基準

- 機能評価のpass@3 > 90%
- リグレッション評価のpass^3 = 100%
```

2. ユーザーに具体的な基準を記入するよう促す

## 評価のチェック

`/eval check feature-name`

機能の評価を実行:

1. `.claude/evals/feature-name.md`から評価定義を読み取り
2. 各機能評価について:
   - 基準を検証しようと試行
   - パス/失敗を記録
   - 試行を`.claude/evals/feature-name.log`に記録
3. 各リグレッション評価について:
   - 関連テストを実行
   - ベースラインと比較
   - パス/失敗を記録
4. 現在のステータスを報告:

```
評価チェック: feature-name
========================
機能: X/Y パス中
リグレッション: X/Y パス中
ステータス: 進行中 / 準備完了
```

## 評価レポート

`/eval report feature-name`

包括的な評価レポートを生成:

```
評価レポート: feature-name
=========================
生成日: $(date)

機能評価
----------------
[eval-1]: パス (pass@1)
[eval-2]: パス (pass@2) - リトライ必要
[eval-3]: 失敗 - ノートを参照

リグレッション評価
----------------
[test-1]: パス
[test-2]: パス
[test-3]: パス

メトリクス
-------
機能 pass@1: 67%
機能 pass@3: 100%
リグレッション pass^3: 100%

ノート
-----
[問題点、エッジケース、観察事項]

推奨
----------
[リリース可 / 要作業 / ブロック中]
```

## 評価一覧

`/eval list`

すべての評価定義を表示:

```
評価定義
===============
feature-auth   [3/5 パス中]  進行中
feature-search [5/5 パス中]  準備完了
feature-export [0/4 パス中]  未開始
```

## 引数

$ARGUMENTS:

- `define <n>` - 新しい評価定義を作成
- `check <n>` - 評価を実行してチェック
- `report <n>` - 完全なレポートを生成
- `list` - すべての評価を表示
- `clean` - 古い評価ログを削除（最新10回を保持）
