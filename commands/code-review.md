# コードレビュー

コミットされていない変更に対する包括的なセキュリティと品質レビュー:

1. 変更されたファイルを取得: `git diff --name-only HEAD`

2. 各変更ファイルについて、以下をチェック:

**セキュリティ問題（クリティカル）:**

- ハードコードされた認証情報、APIキー、トークン
- SQLインジェクション脆弱性
- XSS脆弱性
- 入力検証の欠如
- 安全でない依存関係
- パストラバーサルリスク

**コード品質（高）:**

- 50行を超える関数
- 800行を超えるファイル
- 4レベルを超えるネストの深さ
- エラーハンドリングの欠如
- console.log文
- TODO/FIXMEコメント
- パブリックAPIのJSDocの欠如

**ベストプラクティス（中）:**

- ミューテーションパターン（代わりにイミュータブルを使用）
- コード/コメントでの絵文字の使用
- 新しいコードのテスト欠如
- アクセシビリティの問題（a11y）

3. レポートを生成:
   - 重大度: クリティカル、高、中、低
   - ファイルの場所と行番号
   - 問題の説明
   - 推奨される修正

4. クリティカルまたは高の問題がある場合はコミットをブロック

セキュリティ脆弱性のあるコードは絶対に承認しない！
