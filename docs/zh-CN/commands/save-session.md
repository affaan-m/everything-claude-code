# 保存会话命令

捕获本次会话中发生的一切——构建了什么、什么成功了、什么失败了、还有哪些遗留事项——并将其写入一个带日期的文件，以便下次会话能从此处继续。

## 使用时机

* 在关闭 Claude Code 之前，工作会话结束时
* 在达到上下文限制之前（先运行此命令，然后开始一个新会话）
* 解决了一个想要记住的复杂问题之后
* 任何需要将上下文移交给未来会话的时候

## 流程

### 步骤 1：收集上下文

在写入文件之前，收集：

* 读取本次会话期间修改的所有文件（使用 git diff 或从对话中回忆）
* 回顾讨论、尝试和决定的内容
* 记录遇到的任何错误及其解决方法（或未解决的情况）
* 如果相关，检查当前的测试/构建状态

### 步骤 2：如果不存在则创建会话文件夹

在用户的 Claude 主目录中创建规范的会话文件夹：

```bash
mkdir -p ~/.claude/session-data
```

### 步骤 3：写入会话文件

创建 `~/.claude/session-data/YYYY-MM-DD-<short-id>-session.tmp`，使用今天的实际日期和一个满足 `SESSION_FILENAME_REGEX` 在 `session-manager.js` 中强制执行的规则的短标识符：

* 兼容字符：字母 `a-z` / `A-Z`、数字 `0-9`、连字符 `-`、下划线 `_`
* 兼容最小长度：1 个字符
* 新文件的推荐风格：小写字母、数字和连字符，长度 8 个字符以上以避免冲突

有效示例：`abc123de`、`a1b2c3d4`、`frontend-worktree-1`、`ChezMoi_2`
新文件应避免：`A`、`test_id1`、`ABC123de`

完整有效文件名示例：`2024-01-15-abc123de-session.tmp`

旧文件名 `YYYY-MM-DD-session.tmp` 仍然有效，但新的会话文件应首选短 ID 形式，以避免同一天的冲突。

### 步骤 4：用以下所有部分填充文件

诚实地写入每个部分。不要跳过任何部分——如果某个部分确实没有内容，则写“Nothing yet”或“N/A”。一个不完整的文件比诚实的空部分更糟糕。

### 步骤 5：向用户展示文件

写入后，显示完整内容并询问：

```
会话已保存至 [实际解析的会话文件路径]

这看起来准确吗？在关闭之前，还有什么需要纠正或补充的吗？
```

等待确认。如果用户要求，进行编辑。

## 我们正在构建的内容

Next.js应用的用户认证系统。用户通过邮箱/密码注册，
接收存储在httpOnly Cookie（而非localStorage）中的JWT，受保护的路由
通过中间件检查有效令牌。目标是在浏览器刷新时保持会话持久性，
同时不将令牌暴露给JavaScript。

---

## 已成功实现的内容（附证据）

- **`/api/auth/register` 端点** — 确认方式：Postman POST请求返回200状态码及用户对象，
  Supabase仪表板中可见数据行，bcrypt哈希正确存储
- **`lib/auth.ts` 中的JWT生成** — 确认方式：单元测试通过
  （`npm test -- auth.test.ts`），在jwt.io解码令牌显示正确载荷
- **密码哈希** — 确认方式：测试中`bcrypt.compare()`返回true

---

## 未成功的内容（及原因）

- **Next-Auth库** — 失败原因：与我们自定义的Prisma适配器冲突，
  每次请求都抛出"无法在此配置中将适配器与凭据提供程序一起使用"错误。
  不值得调试——对我们的设置来说过于固执己见。
- **将JWT存储在localStorage中** — 失败原因：SSR渲染发生在localStorage可用之前，
  导致每次页面加载时出现React水合不匹配错误。
  这种方法与Next.js SSR根本不相容。

- 使用`cookies()`从`next/headers`在服务器组件中读取令牌
- 编写middleware.ts通过检查Cookie存在性来保护路由

## 注意事项

* 每个会话都有自己的文件——切勿追加到之前会话的文件中
* "未成功的方法"部分最为关键——如果没有它，未来的会话将盲目重试失败的方法
* 如果用户要求在会话中途保存（不仅仅是结束时），请保存已知内容并清晰标记进行中的项目
* 该文件旨在供 Claude 在下一个会话开始时通过 `/resume-session` 读取
* 使用规范的全局会话存储：`~/.claude/session-data/`
* 对于任何新的会话文件，优先使用短标识符文件名形式（`YYYY-MM-DD-<short-id>-session.tmp`）
