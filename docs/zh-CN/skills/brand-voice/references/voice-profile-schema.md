# 语音档案架构

构建可复用的语音档案时，请严格遵循以下结构：

```text
声音档案
=============
作者：
目标：
信心度：

来源集
- 来源 1
- 来源 2
- 来源 3

节奏
- 关于句子长度、节奏和碎片化的简短说明

压缩度
- 写作的密度或解释性程度

大小写
- 常规、混合或视情况而定

括号使用
- 如何使用以及如何不使用

问题使用
- 罕见、频繁、修辞性、直接或基本缺失

主张风格
- 主张如何构建、支撑和强化

偏好手法
- 作者确实使用的具体手法

禁用手法
- 作者不使用的特定模式

行动号召规则
- 如何、何时或是否以要求结尾

渠道说明
- X：
- LinkedIn：
- 电子邮件：
```

指南：

* 保持档案具体且基于来源。
* 使用简短要点，而非长篇段落。
* 每个禁止行为均需在来源集中可观察，或由用户明确要求。
* 若来源集存在冲突，应明确指出分歧点，而非折中处理成模糊表述。
