# 浏览器QA —— 自动化视觉测试与交互

## 使用场景

* 将功能部署到预发布/预览环境后
* 需要跨页面验证UI行为时
* 发布前确认布局、表单、交互功能正常
* 审查涉及前端代码的PR时
* 无障碍审计和响应式测试

## 工作原理

利用浏览器自动化MCP（claude-in-chrome、Playwright或Puppeteer）像真实用户一样与实时页面交互。

### 第一阶段：冒烟测试

```
1. 导航至目标 URL
2. 检查控制台错误（过滤干扰项：分析工具、第三方）
3. 确认网络请求中无 4xx/5xx 错误
4. 在桌面端和移动端视口截取首屏截图
5. 检查核心网页指标：LCP < 2.5秒，CLS < 0.1，INP < 200毫秒
```

### 第二阶段：交互测试

```
1. 点击每个导航链接 — 验证无死链
2. 使用有效数据提交表单 — 验证成功状态
3. 使用无效数据提交表单 — 验证错误状态
4. 测试认证流程：登录 → 受保护页面 → 退出
5. 测试关键用户旅程（结账、引导、搜索）
```

### 第三阶段：视觉回归

```
1. 在 3 个断点（375px、768px、1440px）截取关键页面截图
2. 与基准截图（如有存储）进行对比
3. 标记布局偏移超过 5px、元素缺失、溢出问题
4. 如适用，检查深色模式
```

### 第四阶段：无障碍测试

```
1. 在每个页面上运行 axe-core 或等效工具
2. 标记 WCAG AA 违规项（对比度、标签、焦点顺序）
3. 验证键盘导航是否端到端正常工作
4. 检查屏幕阅读器地标
```

## 输出格式

```markdown
## QA报告 — [URL] — [时间戳]

### 冒烟测试
- 控制台错误：0个严重错误，2个警告（分析噪声）
- 网络：全部200/304，无失败
- 核心网页指标：LCP 1.2秒 ✓，CLS 0.02 ✓，INP 89毫秒 ✓

### 交互
- [✓] 导航链接：12/12正常工作
- [✗] 联系表单：缺少无效邮箱的错误状态
- [✓] 认证流程：登录/登出正常工作

### 视觉
- [✗] 英雄区域在375px视口下溢出
- [✓] 深色模式：所有页面一致

### 无障碍
- 2项AA违规：英雄图片缺少替代文本，页脚链接对比度低

### 结论：修复后发布（2个问题，0个阻塞项）
```

## 集成方式

适用于任何浏览器MCP：

* `mChild__claude-in-chrome__*` 工具（推荐 —— 使用您实际的Chrome浏览器）
* 通过 `mcp__browserbase__*` 使用Playwright
* 直接使用Puppeteer脚本

配合 `/canary-watch` 进行部署后监控。
