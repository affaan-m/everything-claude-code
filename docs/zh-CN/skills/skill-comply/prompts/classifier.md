您正在对来自编码代理会话的工具调用进行分类，以匹配预期的行为步骤。

对于每个工具调用，确定它属于哪个步骤（如果有）。一个工具调用最多只能匹配一个步骤。

步骤：
{steps\_description}

工具调用（编号）：
{tool\_calls}

仅返回一个 JSON 对象，将 step\_id 映射到匹配的工具调用编号列表。
仅包含至少有一个匹配的步骤。如果没有工具调用匹配某个步骤，则省略它。

示例响应：
{"write\_test": \[0, 1], "run\_test\_red": \[2], "write\_impl": \[3, 4]}

规则：

* 基于工具调用的含义进行匹配，而不仅仅是关键词
* 写入 "test\_calculator.py" 是测试文件写入，即使内容类似于实现
* 写入 "calculator.py" 是实现写入，即使包含测试辅助函数
* 运行 "pytest" 并输出 "FAILED" 的 Bash 命令是 RED 阶段测试运行
* 运行 "pytest" 并输出 "passed" 的 Bash 命令是 GREEN 阶段测试运行
* 每个工具调用最多匹配一个步骤（选择最佳匹配）
* 如果工具调用不匹配任何步骤，则不包含它
