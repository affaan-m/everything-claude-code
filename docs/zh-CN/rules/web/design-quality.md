# Web 设计质量标准

## 反模板政策

不得交付外观通用的模板化 UI。前端输出应具有目的性、独特风格，并体现产品特性。

### 禁止模式

* 间距统一、无层级感的默认卡片网格
* 带有居中标题、渐变色块和通用行动号召的模板化英雄区域
* 未经修改的库默认样式冒充完成设计
* 无层次、深度或动效的扁平布局
* 所有组件使用统一的圆角、间距和阴影
* 安全灰底白字搭配单一装饰色
* 侧边栏+卡片+图表、缺乏设计观点的仪表盘式布局
* 无明确理由使用的默认字体堆栈

### 必备品质

每个有意义的前端界面应至少体现以下四项：

1. 通过尺寸对比建立清晰层级
2. 间距具有设计节奏，而非统一填充
3. 通过重叠、阴影、表面或动效营造深度或层次
4. 具有特色的字体搭配策略
5. 色彩具有语义性，而非仅作装饰
6. 悬停、聚焦和激活状态经过精心设计
7. 适当时采用打破网格的编辑式或便当盒式布局
8. 符合视觉方向时使用纹理、颗粒或氛围效果
9. 动效能清晰引导流程而非分散注意力
10. 数据可视化作为设计系统的一部分，而非事后补充

## 编写前端代码前

1. 选择明确的设计方向。避免"简洁极简"等模糊默认项。
2. 有目的地定义调色板。
3. 精心选择字体。
4. 至少收集少量真实参考案例。
5. 在相关情况下运用 ECC 设计/前端技能。

## 值得尝试的设计方向

* 编辑式/杂志风格
* 新粗野主义
* 具有真实深度的玻璃拟态
* 暗调奢华或明调奢华（严谨对比）
* 便当盒布局
* 滚动叙事
* 3D 集成
* 瑞士/国际风格
* 复古未来主义

不要默认启用深色模式。根据产品实际需求选择视觉方向。

## 组件检查清单

* \[ ] 是否避免看起来像默认的 Tailwind 或 shadcn 模板？
* \[ ] 是否具有经过设计的悬停/聚焦/激活状态？
* \[ ] 是否使用层级而非统一强调？
* \[ ] 在真实产品截图中是否具有可信度？
* \[ ] 若支持双主题，浅色和深色模式是否都显得有目的性？
